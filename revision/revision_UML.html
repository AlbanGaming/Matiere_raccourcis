Intro COO & UML
Pourquoi la conception objet ? Qu'est-ce qu'UML ?

L'approche objet repose sur une idée simple : le monde réel = ensemble d'objets qui collaborent.


Avant (approche fonctionnelle) : une application = un ensemble de fonctions. Le système était vu comme un flux de traitements.


Avec la COO :

  • Un système = des objets ayant des données (attributs) et des comportements (méthodes)
  • Les objets communiquent entre eux en s'envoyant des messages
  • Mécanismes clés : Encapsulation, Modularité, Abstraction, Polymorphisme, Héritage

Avantages : réutilisabilité, évolutivité, modélisation plus proche du monde réel.

Unified Modeling Language = Langage de Modélisation Objet Unifié


UML est un langage (pas une méthode) :

  • Composé d'un ensemble de notations graphiques
  • Pour représenter un système informatique et son utilisation prévue
  • Indépendant des langages de programmation
  • Standard OMG (Object Management Group) — fondé par Booch, Jacobson, Rumsbaugh

3 modes d'utilisation :

Esquisse

Diagrammes informels, incomplets, dessinés à la main. Communication entre équipes.

Plan

Diagrammes de conception détaillés. Ingénierie et reverse-engineering.

Programmation

Génération automatique de code depuis UML (Round-trip engineering).

🔷 Agrégation

Relation de contenance. Le composant peut exister sans le conteneur.

Symbole : losange vide côté conteneur

Page ◇────── Mot
(conteneur)    (composant)

Si on supprime la Page, les Mots peuvent continuer d'exister.

🔶 Composition

Agrégation forte. Lien de vie : si le composite est détruit, les composants aussi.

Symbole : losange plein côté composite

Livre ◆────1..* Page
(composite)    (composant)

Si on supprime le Livre, ses Pages sont supprimées. Le composite est responsable du composant.

Multiplicités — Format

Combien d'instances de B connaît une instance de A ?

NotationSignificationExemple
nExactement nExactement 3
n,m ou n..pExactement n ou m ou p1,3 ou 1..3
n..mEntre n et m (inclus)2..5
n..*Au moins n1..* (au moins 1)
0..nAu plus n0..3
*Plusieurs (0 ou plus)0..*
0..1Optionnel (0 ou 1)Peut en avoir 1 ou aucun

⚠️ La multiplicité se lit du côté opposé : la multiplicité côté B indique combien de B une instance de A connaît.

Navigabilité

Indique dans quel sens on peut "naviguer" (accéder) d'une classe à l'autre.

Bidirectionnelle (défaut)

Personne ←────→ Voiture : on peut accéder de chaque côté

En PHP : Personne a un attribut mesTutures[] ET Voiture a un attribut propriétaire

Unidirectionnelle

Personne ─────▶ Voiture : Personne connaît Voiture, mais pas l'inverse

En PHP : Personne a mesTutures[], mais Voiture n'a PAS d'attribut propriétaire

Aucune direction

Personne ──X── Voiture : aucun passage possible (rare)

Traduction en code (PHP)

Pour Personne 1 ──possède──▶ 0..* Voiture :

private string $immat; (propriétaire non stocké)

Héritage & Classes Abstraites
Généralisation / Spécialisation — Polymorphisme — Override

Relation de Généralisation / Spécialisation :

  • Une relation implique qu'un des deux concepts est une factorisation de l'autre
  • Classe mère = super-classe = classe générale
  • Classe fille = sous-classe = classe spécialisée
  • La classe fille est une sorte de classe mère

Représentation UML — triangle vide côté mère

        A
        △
    ────────
   /         \
  B          C

B et C héritent de A. "B est un A", "C est un A"

En PHP : class B extends A { ... }
class C extends A { ... }
Implémentation TCI :

Tableau Classe/Implém : liste les attributs et méthodes de chaque classe dans la hiérarchie.

Classe abstraite = classe qu'on ne peut pas instancier directement. Elle sert de modèle.

Notation UML : nom en italique ou stéréotype <<abstract>>


Méthode abstraite = méthode déclarée mais sans corps. Les classes filles doivent l'implémenter.

Notation UML : méthode en italique


Étapes de modélisation :
1. "Est une sorte de" ?
2. Factoriser / spécialiser
3. Vérifier l'encapsulation
4. Comportements différents par classe → polymorphisme + override

<<abstract>>MoyenTransport
- immat : String
+ getImmat() : String
+ seDeplacer() : String {abstract}
△
Bateau
+ seDeplacer() : String {override}
Avion
+ seDeplacer() : String {override}

PHP : abstract class MoyenTransport + abstract public function seDeplacer()

Polymorphisme = même appel de méthode → comportement différent selon l'objet réel.

$t = new Avion();
echo $t->seDeplacer(); // retourne "air"

$t = new Bateau();
echo $t->seDeplacer(); // retourne "eau"

Override (redéfinition) = la classe fille réécrit une méthode héritée. En UML : {override}. En PHP : simplement redéclarer la méthode avec le même nom.

<<interface>> — Définition

  • Classe sans attributs où toutes les méthodes sont abstraites
  • Ne peut pas être instanciée
  • Doit être réalisée (implémentée) par une classe non abstraite
  • Relation de réalisation : flèche pointillée avec triangle (- - -▷)

Notation UML

  public function voler();
}

class CanardColVert implements Volant {
  public function voler() { ... }
}

💡 Avantage : une classe peut implémenter plusieurs interfaces (contrairement à l'héritage simple). Principe "Utiliser une interface, pas une réalisation".

Éléments d'un diagramme de séquence

Acteur / Instance : représenté par un rectangle en haut (ligne de vie). Nom : nom : ClasseA (souligné).


Ligne de vie : trait vertical pointillé qui descend sous l'acteur. Représente son existence dans le temps.


Activation : rectangle fin sur la ligne de vie = période pendant laquelle l'objet est actif (exécute quelque chose).


Création : flèche pointant vers le rectangle de l'instance créée. Destruction : croix (X) en bas de la ligne de vie.

Message synchrone

L'émetteur attend la réponse avant de continuer. Flèche pleine (─────▶). C'est le type par défaut.

A ─────────▶ B

Message asynchrone

L'émetteur n'attend pas la réponse. Flèche ouverte (──────▷).

A ──────────▷ B

Message de retour

Réponse à un message. Flèche pointillée (- - - -▶). Peut être omis si implicite.

B - - - - -▶ A

Message réflexif

Un objet s'envoie un message à lui-même. Boucle sur la même ligne de vie.

A ─┐
   └─▶ A (calcul interne)
