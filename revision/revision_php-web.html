HTTP, Cookies & Sessions
Protocole du Web — échanges client/serveur, gestion d'état.

🌐 Protocole HTTP

HTTP (HyperText Transfer Protocol) est un protocole d'échange orienté lignes de caractères entre un client (navigateur) et un serveur Web. Il est sans état : le serveur oublie le client après chaque transaction.

Versions

VersionCaractéristiques
HTTP/1.01 transaction = 1 ressource. Connexion fermée après chaque échange.
HTTP/1.1Connexion persistante, cache standardisé, header Host: obligatoire, 1 IP = N serveurs virtuels.

Structure d'un message

// Requête ou état (1 ligne)
// En-têtes : valeur
// Ligne vide (fin des en-têtes)
// Corps (optionnel)

Méthodes HTTP

MéthodeUsage
GETDemande la ressource + ses infos
POSTEnvoie des données (formulaire) + demande la ressource
HEADInfos sur la ressource uniquement (sans corps)
PUT/PATCH/DELETEAPI Web (REST)

Codes de réponse

ClasseCodeSignification
2xx200 OKSuccès
3xx304 NOT MODIFIEDRedirection / cache
4xx403 FORBIDDEN, 404 NOT FOUNDErreur client
5xx500 INTERNAL ERRORErreur serveur

Formulaires

methodParamètres dansUsage
GETURL (query string)Recherche, bookmark
POSTCorps du messageDonnées sensibles, fichiers

🍪 Cookies

Les cookies permettent au serveur de « se souvenir » du client entre les requêtes.

Serveur → Client via header Set-Cookie: var=val; expires=…; path=…; domain=…
Client → Serveur via header Cookie: var=val

Avantages / Inconvénients

  • ✓ Pas besoin de modifier les pages HTML
  • ✓ Durée de validité configurable
  • ✗ Données en clair sur le réseau
  • ✗ Données transmises à chaque requête

setcookie() en PHP

setcookie(
  'nom',          // nom du cookie
  'valeur',       // valeur
  time() + 864000, // expiration (+10 jours)
  '/',            // path (tout le serveur)
  'example.com', // domaine
  false,          // secure (HTTPS only ?)
  true            // httponly
);

Lecture

// Superglobal $_COOKIE
if (isset($_COOKIE['passage'])) { ... }

// Supprimer un cookie
setcookie('passage', false);
⚠️ setcookie() doit être appelé avant tout echo — il envoie un header HTTP.

🔐 Sessions

Les sessions stockent les données côté serveur. Seul l'identifiant de session (SHA-2 512 bits) circule côté client via cookie.

Fonctions principales

session_start();         // crée ou restaure une session
$_SESSION['user'] = 'Alice'; // stocker une donnée
$id = session_id();      // lire l'identifiant de session
session_unset();          // détruire toutes les variables
session_destroy();        // détruire la session

session_set_cookie_params()

session_set_cookie_params(
  10*24*60*60,  // lifetime (secondes)
  '/',           // path
  'example.com', // domain
  false,         // secure
  true           // httponly
);
💡 Différence clé : les sessions stockent tout côté serveur. Les cookies stockent les données côté client.
PHP — Bases
Langage de script serveur, interprété, faiblement typé.

🐘 Qu'est-ce que PHP ?

  • Créé en 1994-1995 par Rasmus Lerdorf
  • Acronyme récursif : PHP: Hypertext Preprocessor
  • Utilisé par ≈72% des serveurs Web (2026)
  • Versions actuelles : 8.2, 8.3, 8.4, 8.5
  • Cycle : 2 ans support actif + 1 an sécurité

Caractéristiques

PropriétéValeur
ParadigmeImpératif, orienté objet possible
TypageFaible (coercition automatique)
ExécutionInterprété côté serveur
PlateformesUNIX, Windows, macOS

⚙️ Fonctionnement en mode Web

Le client envoie une requête HTTP → le serveur Web localise le fichier .php → le module PHP l'exécute → le résultat HTML est renvoyé au client.

// Séquence typique :
// 1. GET /hello.php HTTP/1.0
// 2. Localisation de hello.php
// 3. Exécution du code PHP
// 4. Réponse : 200 OK + HTML généré

<?php
echo "<html><body>Hello world</body></html>";
// La page HTML est construite à la volée
Le code PHP n'est jamais envoyé au client. Seul le résultat HTML/CSS/JS l'est.

📝 Définir et utiliser une classe

class MyClass
{
    private string $myData;

    public function __construct(string $data) {
        $this->myData = $data;
    }

    public function __destruct() {
        echo 'Destruction...';
    }

    public function show(): void {
        echo 'my data : ' . $this->myData;
    }
}

// Utilisation
$object = new MyClass('test');
$object->show(); // my data : test

📌 Attributs et méthodes de classe (static)

  • Mot-clé static : utilisable sans instance
  • Attributs partagés par toutes les instances
  • Dans les méthodes statiques : $this non disponible
  • Accès : Classe::$attribut ou Classe::methode()
class MyClass {
    private static int $n = 0;
    public function __construct() { $n++; }
    public static function getCount(): int {
        return self::$n;
    }
}
echo MyClass::getCount(); // 0

🔗 Héritage

class Simple {
    public function show(): void { echo 'Je suis Simple'; }
}

class Extended extends Simple {
    public function show(): void {
        parent::show(); // appel méthode parent
        echo ' mais aussi Extended';
    }
}
PHP ne supporte que l'héritage simple (une seule classe mère). Pour réutiliser du code de plusieurs sources → utiliser les Traits.

🎭 Traits

Mécanisme de réutilisation de code horizontale (composition). Un trait contient des méthodes incluses dans plusieurs classes sans relation d'héritage.

trait Loggable {
    public function log(string $msg): string {
        return date('H:i:s') . ' - ' . $msg;
    }
}

class User {
    use Loggable; // intègre le trait
    public function __construct(public string $name) {}
}

$u = new User('Alice');
echo $u->log('Info'); // 10:33:11 - Info

⚠️ Exceptions

Mot-cléRôle
throwJette une exception (arrête l'exécution locale)

catch (Type $e) { }Capture l'exception
try {
    throw new Exception('Mon erreur');
    echo 'Jamais exécuté';
}
catch (Exception $e) {
    echo $e->getMessage(); // Mon erreur
}

// Multi-catch PHP 8
catch (InvalidArgumentException | RuntimeException $e) { ... }

Méthodes utiles de Exception

  • getMessage() — message d'erreur
  • getCode() — code numérique
  • getFile() — fichier source
  • getLine() — ligne du fichier
  • getTrace() — tableau de backtrace

🔄 Assignation vs Clonage

⚠️ En PHP, affecter un objet ($b = $a) ne le copie pas ! Les deux variables pointent vers le même objet.
// Assignation → même objet
$segment[] = $point;        // référence partagée

// Clonage → copie indépendante
$segment[] = clone $point;  // objet distinct

Passage d'objet en paramètre

Les objets sont passés par référence implicitement : modifier l'objet dans la fonction modifie l'original.

Autoload & Namespace
Organisation des classes, chargement automatique, espaces de noms.

🔄 Auto-chargement de classes

Déclenché quand une classe utilisée n'est pas encore définie — dernière chance d'inclure sa définition.

// autoload.php
spl_autoload_register(
    function (string $class) {
        require_once __DIR__ . "/src/{$class}.php";
    }
);

// Dans le programme (jamais dans une classe !)
require_once 'autoload.php';
$p = new Point(); // déclenche require_once 'src/Point.php'
⚠️ L'autoload ne doit être configuré que dans les programmes, jamais dans les classes (qui doivent rester indépendantes).

🗂️ Espaces de noms (namespace)

Évitent les collisions de noms et regroupent classes/fonctions/constantes dans un espace logique (comme des dossiers pour les fichiers).

// src/Prj/MySQL/Connection.php
namespace Prj\MySQL;
class Connection { /* ... */ }

// src/Prj/LDAP/Connection.php
namespace Prj\LDAP;
class Connection { /* ... */ }

// Utilisation avec alias
use Prj\MySQL\Connection as DBConnection;
use Prj\LDAP\Connection as LDAPConnection;

$mysql = new DBConnection();
$ldap  = new LDAPConnection();
Convention : le chemin du fichier correspond au namespace.
Prj\MySQL\Connection → src/Prj/MySQL/Connection.php

📦 Auto-chargement avec namespace

spl_autoload_register(
  function (string $className) {
    // Remplace \ par / pour le chemin
    $fileName = implode('/', explode('\\', $className));
    $path = __DIR__ . '/src/' . $fileName . '.php';
    if (file_exists($path)) {
        require_once $path;
    }
  }
);

Composer

Outil de gestion des dépendances. Configure l'autoload via composer.json. Génère vendor/autoload.php à inclure une seule fois.

// À inclure UNE seule fois dans le programme
require_once 'vendor/autoload.php';
// Toutes les classes des vendor + src sont disponibles
PDO — PHP Data Objects
Interface orientée objet pour accéder aux bases de données.

🗄️ Présentation

  • Extension PHP fournissant une interface d'abstraction d'accès aux données
  • N'abstrait PAS le SQL (qui reste spécifique au moteur)
  • Interface orientée objet
  • Supporte : MySQL, PostgreSQL, Oracle, SQLite, SQL Server…

3 classes principales

ClasseRôle
PDOConnexion à la base de données
PDOStatementRequête préparée, jeu de résultats
PDOExceptionException spécifique (hérite de Exception)

🔌 Connexion

// DSN : "driver:host=...;dbname=...;charset=..."
$dbh = new PDO(
    'mysql:host=localhost;dbname=ma_base;charset=utf8',
    $user,
    $pass
);

// Fermeture de connexion
$dbh = null;

Gestion des erreurs de connexion

try {
    $dbh = new PDO('mysql:host=dbs;dbname=music', $user, $pass);
    $dbh->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    // ... requêtes ...
    $dbh = null;
} catch (PDOException $e) {
    echo '<p>Erreur de base de données';
    die();
}

Modes d'erreur

ModeComportement
ERRMODE_SILENTSilencieux (défaut) — errorCode() / errorInfo()
ERRMODE_WARNINGÉmet un E_WARNING
ERRMODE_EXCEPTIONLance une PDOException Recommandé

📋 Requêtes préparées

Une requête préparée est analysée + compilée + optimisée une fois, puis exécutée autant de fois que nécessaire avec des paramètres différents.

⚠️ query() est à bannir en production. Utiliser toujours prepare()/execute().

Paramètres anonymes (?)

$stmt = $pdo->prepare('SELECT * FROM user WHERE login = ?');
$stmt->bindValue(1, 'root');      // position 1
$stmt->execute();

Paramètres nommés (:nom)

$stmt = $pdo->prepare(
    'SELECT * FROM user WHERE login = :login'
);
$stmt->bindValue(':login', 'root');
$stmt->execute();

// Ou directement dans execute()
$stmt->execute([':login' => 'root']);

📊 Modes de récupération (fetch)

ConstanteRetourneAccès
FETCH_ASSOCTableau associatif$row['nom_colonne']
FETCH_NUMTableau indexé$row[0]
FETCH_BOTHLes deux (défaut)Les deux
FETCH_OBJObjet anonyme$row->nom_colonne
FETCH_CLASSInstance d'une classe$row->attribut

Parcourir les résultats

// Méthode 1 : foreach direct (itérable)
foreach ($stmt as $row) { echo $row['name']; }

// Méthode 2 : fetchAll (tableau complet)
foreach ($stmt->fetchAll(PDO::FETCH_ASSOC) as $row) { ... }

// Méthode 3 : while + fetch
while (($row = $stmt->fetch(PDO::FETCH_ASSOC)) !== false) {
    echo $row['name'];
}

FETCH_CLASS — instanciation automatique

$stmt = $pdo->query('SELECT id, name FROM artist WHERE id = 12');
$stmt->setFetchMode(PDO::FETCH_CLASS, Artist::class);
$artist = $stmt->fetch(); // objet Artist avec $id, $name

// Ou avec fetchObject()
$artist = $stmt->fetchObject(Artist::class);
Pattern Singleton — MyPDO
Connexion unique à la base de données via le pattern Singleton.

🔒 Problème

Dans un site orienté objet, plusieurs méthodes peuvent avoir besoin d'accéder à la base. Quand se connecter ? Se déconnecter ? Comment éviter les connexions multiples ?

💡 Solution : Singleton

  • Objet à instance unique
  • 1 attribut static qui stocke l'instance
  • 1 méthode static getInstance() qui la retourne (et la crée si nécessaire)
  • Constructeur private (interdit la construction directe)
  • __clone() désactivé

🏗️ MyPDO extends PDO

final class MyPdo extends PDO {
    private static self $myPdoInstance; // instance unique
    private static string $dsn;
    private static string $username = '';
    private static string $password = '';
    private static array $options = [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    ];

    private function __construct(string $dsn, ...) {
        parent::__construct($dsn, ...);
    }

    public static function getInstance(): self {
        if (!isset(self::$myPdoInstance)) {
            self::$myPdoInstance = new self(self::$dsn, ...);
        }
        return self::$myPdoInstance;
    }

    public static function setConfiguration(
        string $dsn, string $username='', string $password=''
    ): void {
        self::$dsn      = $dsn;
        self::$username = $username;
        self::$password = $password;
    }
}

🚀 Utilisation

// Configuration (une seule fois, fichier .ini conseillé)
MyPDO::setConfiguration(
    'mysql:host=mysql;dbname=ma_base;charset=utf8',
    'user', 'pass'
);

// Utilisation : getInstance() crée la connexion si besoin
$stmt = MyPDO::getInstance()->prepare(<<);
$stmt->execute();

while (($row = $stmt->fetch()) !== false) {
    echo "<p>{$row['name']}";
}
// Déconnexion automatique à la fin du script

Durée de vie

ÉvénementMoment
Connexion (construction PDO)Premier appel à getInstance()
Déconnexion (destruction PDO)Fin du script PHP
La configuration peut être externalisée dans un fichier .mypdo.ini pour séparer identifiants et code.
Bibliothèque GD — Images en PHP
Créer, manipuler et envoyer des images (GIF, JPEG, PNG) en PHP.

🖼️ Algorithme général

<?php
// 1. Créer l'image
$im = imageCreateTrueColor(100, 100);

// 2. Allouer des couleurs
$red = imageColorAllocate($im, 255, 0, 0);

// 3. Dessiner
imageFilledRectangle($im, 0, 0, 99, 99, $red);

// 4. Envoyer au navigateur
header('Content-Type: image/png');
imagePNG($im);

// 5. Libérer la mémoire
imageDestroy($im);

🔧 Fonctions principales

Créer une image vide

FonctionType d'image
imageCreate($w, $h)Couleurs indexées (palette)
imageCreateTrueColor($w, $h)Couleurs réelles (24 bits) Recommandé
imageDestroy($im)Libère la mémoire

Créer depuis un fichier

FonctionFormat
imageCreateFromGIF($nom)GIF
imageCreateFromJPEG($nom)JPEG
imageCreateFromPNG($nom)PNG

Produire l'image

// Vers un fichier
imagePNG($im, 'output.png');

// Vers le navigateur (sans 2ème arg + header Content-Type)
header('Content-Type: image/png');
imagePNG($im);

Couleurs

// RGB
$c = imageColorAllocate($im, 255, 128, 0);

// RGBA (alpha 0=opaque, 127=transparent)
$c = imageColorAllocateAlpha($im, 255, 128, 0, 64);

Dessin

imageArc($im, $cx, $cy, $w, $h, $start, $end, $color);
imageLine($im, $x1, $y1, $x2, $y2, $color);
imageString($im, $font, $x, $y, $texte, $color);

🔄 Images à la volée

// Dans la page HTML :
<img src="img.php?t=Coucou&r=255" alt="Image générée">

// img.php utilise $_GET['t'], $_GET['r'] etc.
Usages typiques : miniatures, CAPTCHA, graphiques, traitements par lots.

Copie redimensionnée

imageCopyResized($dst, $src,
    $dx, $dy, $sx, $sy,
    $dw, $dh, $sw, $sh);

// Mieux (ré-échantillonnage) :
imageCopyResampled(...); // qualité supérieure
