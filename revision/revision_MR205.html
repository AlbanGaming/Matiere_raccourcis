// Enregistrements DNS

Types de records

A — Address ▼
Associe un FQDN à une adresse IPv4.

ns1 IN A 192.168.1.2
AAAA — Address IPv6 ▼
Comme A, mais pour les adresses IPv6.
CNAME — Alias ▼
Définit un alias vers un autre nom canonique.

www IN CNAME ns1
MX — Mail eXchanger ▼
Désigne le(s) serveur(s) de messagerie d'une zone. La priorité (valeur basse = préféré) est obligatoire.
NS — Name Server ▼
Indique les serveurs DNS autoritaires pour la zone.
PTR — Pointer (reverse) ▼
Résolution inverse : adresse IP → nom de domaine. Utilisé dans la zone in-addr.arpa.
SOA — Start Of Authority ▼
Premier enregistrement obligatoire. Définit le serveur primaire, l'email du responsable et les paramètres de synchronisation (serial, refresh, retry, expire, TTL).
TXT — Text ▼
Données texte libres, utilisées notamment pour la vérification SPF, DKIM (anti-spam).

1. DHCP — Dynamic Host Configuration Protocol

DHCP permet l'allocation automatique d'adresses IP à des clients sur un réseau, sans configuration manuelle.

Les 3 types d'allocation

STATIQUE
Toujours la même IP pour un hôte donné (liaison MAC→IP).
DYNAMIQUE TEMPORAIRE
IP avec un bail (lease) limité dans le temps. Redistribuée à expiration.
DYNAMIQUE PERMANENTE
IP "réservée" à un PC. Bail très long = quasi-statique.

Séquence DORA — Obtention d'une adresse

DISCOVER
Client broadcast pour trouver un serveur DHCP
OFFER
Serveur propose une adresse IP
REQUEST
Client accepte et demande officiellement l'IP
ACK
Serveur confirme l'attribution
⚠️
Attention Si client et serveur sont sur des LAN différents, il faut un relais DHCP car les routeurs ne transmettent pas les broadcasts Ethernet !

Configuration Cisco — Pool DHCP

Cisco IOS
! Activation du service
R(config)# service dhcp

! Création d'un pool nommé
R(config)# ip dhcp pool campus
R(dhcp-config)# network 172.16.12.0 255.255.255.0
R(dhcp-config)# default-router 172.16.12.1
R(dhcp-config)# dns-server 10.10.10.1
R(dhcp-config)# domain-name iut-info.test

! Exclure des adresses (passerelle, serveurs...)
R(config)# ip dhcp excluded-address 172.16.12.1 172.16.12.10

! Vérification des baux
R# show ip dhcp binding
💡
À retenir La commande ip dhcp excluded-address doit être saisie avant la création du pool pour éviter que ces IPs soient distribuées.

2. DNS — Domain Name System

Base de données distribuée et hiérarchique qui traduit les noms de domaine en adresses IP (et inversement).

Arborescence DNS

. (racine)
├── fr
│   └── univ-reims
│       └── iut-info  →  iut-info.univ-reims.fr.
├── com
│   └── google
│       └── www  →  www.google.com.
└── org
    └── ietf

Types de TLD

  • Organisationnel : .com .org .edu .gov .mil
  • Géographique : .fr .jp .de .eu
  • Inverse : in-addr.arpa (résolution IP→nom)

Résolution inverse

Pour trouver le nom à partir d'une IP, on inverse les octets et on ajoute .in-addr.arpa :

192.168.1.5  →  5.1.168.192.in-addr.arpa

Serveur primaire vs secondaire

  • Primaire (master) : maintient le fichier de zone, fait autorité.
  • Secondaire : copie les données du primaire (redondance).

Résolution récursive

Application
Demande l'IP de www.example.fr
Résolveur local
Interroge le serveur DNS configuré
Racine .
Délègue vers .fr
Serveur .fr
Délègue vers example.fr
Réponse
IP retournée à l'application
🔍
Outils de diagnostic host, nslookup, dig permettent d'interroger les serveurs DNS manuellement. dig +trace trace le chemin complet de délégation depuis la racine.

3. Configuration BIND sous Linux

named.conf — Fichier principal

named.conf
// Répertoire des fichiers de zone
options {
  directory "/var/named";
};

// Serveurs racine
zone "." {
  type hint;
  file "named.root";
};

// Zone reverse loopback
zone "0.0.127.in-addr.arpa" {
  type master;
  file "localhost.rev";
};

// Zone principale
zone "iut-info.test" {
  type master;
  file "iut-info.test";
};

Fichier de zone directe

iut-info.test (zone file)
$TTL 3600
iut-info.test.  IN  SOA  ns1.iut-info.test.  admin.iut-info.test. (
                          2024010101  ; Serial
                          10800       ; Refresh (3h)
                          3600        ; Retry (1h)
                          604800      ; Expire (7j)
                          86400 )     ; Minimum TTL

        IN  NS    ns1.iut-info.test.
        IN  NS    ns2.iut-info.test.
        IN  MX 10 mail.iut-info.test.
        IN  A     10.10.10.1

ns1     IN  A     10.10.10.1
ns2     IN  A     10.10.10.2
www     IN  A     172.16.12.10
mail    IN  A     10.10.10.3
ftp     IN  CNAME www          ; alias

Fichier de zone reverse

zone reverse (in-addr.arpa)
$TTL 3600
10.10.10.in-addr.arpa.  IN  SOA  ns1.iut-info.test.  admin.iut-info.test. (
                           2024010101 10800 3600 604800 86400 )
                       IN  NS   ns1.iut-info.test.

1  IN  PTR  ns1.iut-info.test.
2  IN  PTR  ns2.iut-info.test.
3  IN  PTR  mail.iut-info.test.
📌
Le SOA en détail Serial : numéro de version (incrémenté à chaque modif) · Refresh : fréquence de synchro secondaire → primaire · Retry : délai de nouvelle tentative · Expire : durée avant abandon · TTL : durée de cache chez les clients

4. FTP — File Transfer Protocol

FTP utilise deux connexions TCP distinctes :

PORT 21 — CONTRÔLE
Commandes client↔serveur (liste, renommage...). Connexion permanente, initiée par le client.
PORT 20 — DONNÉES
Transfert des fichiers. Connexion temporaire, initiée par le serveur, fermée sur EOF.
🔓
FTP = non sécurisé Login et mot de passe transitent en clair ! Utiliser SFTP ou FTP over SSH à la place.

Versions sécurisées

  • SFTP (SSH File Transfer Protocol) : chiffrement complet données + commandes via SSH
  • FTP over SSH : tunnel SSH autour d'une session FTP classique
  • FTPS : FTP avec SSL/TLS (moins courant)

5. SSH — Secure Shell

SSH remplace Telnet et les outils rsh/rlogin en offrant :

  • Authentification par clés RSA (privée + publique)
  • Chiffrement de tous les segments TCP
  • Traversée des pare-feux
  • Création de tunnels TCP

Tunnel SSH

Bash
# Forme générique
ssh [USER@]SSH_SERVER -L [LOCAL_IP:]LOCAL_PORT:DEST_IP:DEST_PORT

# Exemple : accéder à 192.168.1.10:80 via le serveur 1.2.3.4
ssh user@1.2.3.4 -L 2023:192.168.1.10:80

# Puis naviguer en local sur le port du tunnel
w3m http://localhost:2023

Telnet vs SSH

Tunnel❌✅

1. Connexion à distance

Objectif : exécuter des commandes sur un ordinateur distant comme si on était devant lui.

Comparaison des protocoles

ProtocolePortSécuritéParticularités
Telnet23❌ ClairHistorique (1983), mot de passe en clair, obsolète
rlogin513❌ ClairUnix uniquement, "machine de confiance" → faille inhérente
SSH22✅ ChiffréClés RSA, tunnels, remplace Telnet/rlogin/rsh

rlogin — La faille de confiance

⚠️
Faille de sécurité rlogin repose sur la notion de "machine de confiance" : si A fait confiance à B, et B fait confiance à C, alors C peut accéder à A sans authentification. Cette confiance en chaîne est une faille grave.

SSH — Configuration Cisco (TP)

Cisco IOS — Activer SSH
! Prérequis : nom de domaine et hostname
Router(config)# ip domain-name iut-info.test
Router(config)# hostname Router2

! Génération des clés RSA (min 512, recommandé 2048)
Router2(config)# crypto key generate rsa
! → Choose modulus: 768 (ou plus)

! Forcer SSH v2
Router2(config)# ip ssh version 2

! Autoriser SSH sur les lignes VTY
Router2(config)# line vty 0 4
Router2(config-line)# transport input ssh
Router2(config-line)# password class

! Se connecter depuis un client
PC> ssh -l cisco 172.16.1.5
🔐
Pourquoi SSH est sécurisé ? Les credentials ne transitent jamais en clair. L'échange initial établit des clés de session éphémères via Diffie-Hellman. Toutes les trames suivantes sont chiffrées et authentifiées.

2. Accès aux fichiers distants

FTP vs NFS/SMB

CritèreFTPNFS / SMB
UsageCopie intégrale de fichiersAccès transparent, comme local
ModificationTéléchargement puis réuploadÉdition directe sur le serveur
Montage❌✅ Intégré au système de fichiers
DomaineUniverselNFS=Unix, SMB=Windows

NFS — Network File System

Protocole Unix/Linux permettant de monter un répertoire distant comme s'il était local. NFS v4 ajoute le chiffrement Kerberos 5.

SMB / Samba

Protocole Microsoft pour le partage de fichiers et d'imprimantes. Adresse du type :

\\serveur\partage\chemin\nom_fichier

Samba est l'implémentation Linux de SMB, permettant :

  • Partage de disques entre Linux et Windows
  • Services d'impression pour clients Windows
  • Intégration dans un domaine Active Directory
  • Agir comme contrôleur de domaine AD

4. Annuaire LDAP

LDAP organise des informations (utilisateurs, machines, imprimantes) dans une structure hiérarchique appelée DIT (Directory Information Tree).

Différences LDAP vs SGBD

CritèreLDAPSGBD classique
StructureHiérarchique (arbre)Tabulaire (tables)
RelationsAucune dépendanceClés étrangères
SchémaStandardiséPropriétaire
Optimisé pourLecture intensiveLecture/écriture équilibrée

Structure d'une entrée

LDIF
# DN = Distinguished Name (identifiant unique)
dn: cn=John Doe,dc=example,dc=com
cn: John Doe
givenName: John
sn: Doe
telephoneNumber: +33 1 23 45 67 89
mail: john@example.com
manager: cn=Larry Smith,dc=example,dc=com
objectClass: inetOrgPerson
objectClass: person
objectClass: top

Terminologie clé

  • DN (Distinguished Name) : identifiant unique et complet de l'entrée
  • RDN (Relative DN) : partie relative, ex. cn=John Doe
  • dc : Domain Component — composant du domaine
  • cn : Common Name — nom commun
  • ou : Organizational Unit — unité organisationnelle

Principaux serveurs LDAP

  • OpenLDAP — open source, très répandu sous Linux
  • Active Directory (Microsoft) — LDAP + Kerberos
  • 389 Directory Server — RedHat/CentOS

Cisco IOS

! 3. Routes statiques

! 4. Vérification

Ce que fait chaque commande

ip dhcp pool campus▼
Crée un nouveau pool d'adresses DHCP nommé campus. Le nom n'a de signification que locale.
network 172.16.12.0 255.255.255.0▼
Définit la plage d'adresses distribuables : tout le sous-réseau 172.16.12.0/24.
default-router 172.16.12.1▼
Indique la passerelle par défaut que recevront les clients (option DHCP 3).
dns-server 10.10.10.1▼
Communique l'adresse du serveur DNS aux clients (option DHCP 6).
ip dhcp excluded-address▼
Exclut les adresses 172.16.12.1 à 172.16.12.10 du pool pour éviter les conflits avec la passerelle et les serveurs.

Étape 2 : Enregistrements DNS sur Server0

Server0 (10.10.10.1) doit avoir autorité sur iut-info.test. Il faut saisir :

  • Enregistrement SOA : serveur primaire ns.iut-info.test
  • Enregistrements A : ns, www, mail...
  • Enregistrement MX : serveur de messagerie
🎯
Pour accéder à http://www.iut-info.test Il faut : (1) que www.iut-info.test résolve vers 172.16.12.10 dans le DNS, (2) que le serveur HTTP soit actif sur cette machine, (3) que les PC clients aient bien 10.10.10.1 comme serveur DNS.

👁️
Visible dans Wireshark ! En mode simulation Packet Tracer, on peut retrouver le login et le mot de passe en clair dans les trames FTP sur la connexion de contrôle (port 21). C'est la démonstration de l'insécurité de FTP.

SSH sur routeur : succession des trames

  • TCP SYN / SYN-ACK / ACK → établissement connexion TCP port 22
  • SSH banner → échange de version SSH
  • Key Exchange Init → négociation algorithmes
  • Diffie-Hellman → échange de clés
  • New Keys → activation du chiffrement
  • Tout le reste : chiffré → login/password invisibles ✅

TP3 — Installation vsftpd (VM Ubuntu)

Installation et mise à jour

Bash — Ubuntu Server
# Mise à jour du système
sudo apt update && sudo apt upgrade

# Installer SSH (si non coché à l'install)
sudo apt install openssh-server

# Installer vsftpd
sudo apt install vsftpd

# Accès root via sudo
sudo su -

Configuration vsftpd

/etc/vsftpd.conf
# Mode autonome (IPv4 uniquement)
listen=YES
listen_ipv6=NO

# Activer les utilisateurs locaux
local_enable=YES

# Autoriser l'écriture
write_enable=YES

# Connexions anonymes (TP)
anonymous_enable=YES

# Bannière personnalisée
ftpd_banner=Bienvenue sur le serveur FTP IUT !

# Empêcher de sortir du répertoire home
chroot_local_user=YES

Problème chroot + solution

🔒
Erreur avec chroot_local_user=YES vsftpd refuse si le home a des droits d'écriture pour le propriétaire. Solution : créer un sous-dossier et retirer le droit d'écriture sur le home.
Bash — Fix chroot
# Créer un répertoire accessible
mkdir /home/iut/Documents

# Retirer les droits d'écriture du home
chmod a-w /home/iut

# Redémarrer le service
sudo systemctl restart vsftpd

# Vérifier l'état
sudo systemctl status vsftpd
