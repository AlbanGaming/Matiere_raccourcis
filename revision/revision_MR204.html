💡 Chaque équipement agit sur une couche OSI différente — plus il est haut dans les couches, plus il est "intelligent".

Switch en détail

Table MAC : association adresse MAC ↔ port, apprise dynamiquement.

  • Adresse connue → envoi sur le bon port uniquement
  • Adresse inconnue → flooding (broadcast)
  • Même port source = destination → trame ignorée
  • Switch manageable : SNMP, VLANs, QoS, port mirroring, filtrage MAC

Routeur en détail

  • Utilise les adresses IP (couche 3), pas les MAC
  • Routage statique : table configurée manuellement
  • Routage dynamique : auto-apprentissage (OSPF, RIP...)
  • Ne laisse pas passer les broadcasts → délimite les domaines
  • NAT en IPv4 : masque les adresses internes
  • Obligatoire pour connecter un LAN à Internet

Deux types d'adresses

  • MAC (physique) : unique par carte, communication locale uniquement
  • IP (réseau) : identification sur Internet, permet les sous-réseaux

Analogie : rue = réseau, immeuble = IP, boîte aux lettres = port

Adressage IPv4

32 bits = 4 octets — ex: 192.168.1.1

  • Classe A : 0-127.x.x.x → masque /8
  • Classe B : 128-191.x.x.x → masque /16
  • Classe C : 192-223.x.x.x → masque /24
  • CIDR : notation /n (bits réseau), s'affranchit des classes

Adressage IPv6

128 bits — 8 groupes hexadécimaux — ex: 2001:0db8:acad:00c8::1/64

  • GUA : publique routable, commence par 2000::/3
  • LLA : fe80::/10, locale uniquement
  • Préfixe recommandé : /64 (64 bits réseau + 64 bits interface)
  • Élimine le besoin de NAT

Adresse MAC

48 bits = 6 octets — format XX:XX:XX:XX:XX:XX

  • OUI (3 premiers octets) : identifiant fabricant
  • 3 derniers octets : numéro de série unique
  • Unicast : une destination précise
  • Broadcast : FF-FF-FF-FF-FF-FF (tous les équipements)

VLANs

Topologie logique ≠ topologie physique

  • VLAN niveau 1 (par port), niveau 2 (par MAC), niveau 3 (par IP)
  • Séparation sécurité + optimisation bande passante
  • Trunk 802.1Q : un seul câble pour plusieurs VLANs (trames taguées)
  • Pour communiquer entre VLANs → routeur obligatoire

Ethernet (802.3)

CSMA/CD : écoute avant émission, détection de collision → attente aléatoire

Trame Ethernet : Préambule · MAC dest · MAC src · EtherType · Données (46-1500o) · FCS

  • 0x0800 = IPv4 · 0x86DD = IPv6 · 0x0806 = ARP

WiFi (802.11)

CSMA/CA : évite les collisions (pas de détection possible en sans-fil)

Séquence : DIFS → RTS → CTS → Données → ACK

  • 2.4 GHz : canaux recommandés 1, 6, 11 — portée supérieure
  • 5 GHz : moins d'interférences, débit supérieur
  • Connexion : Probe → Authentication → Association
  • Sécurité : WEP (obsolète ❌) · WPA · WPA2-AES (recommandé ✅)

  • Nombre d'hôtes = 2^(bits hôte) − 2

Activité 2 — Questions du TP

Les points essentiels à retenir

Port 80 pour HTTP, port 443 pour HTTPS.
C'est la couche 4 (Transport / TCP) qui gère les ports.
La requête HTTP : méthode GET, l'hôte cible (ex: www.osi.local), la version HTTP. C'est le contenu applicatif pur.
DNS (port 53) — résout www.osi.local → adresse IP
ARP — résout adresse IP → adresse MAC
TCP — établit la connexion (SYN / SYN-ACK / ACK)

Tout ça se passe AVANT même la requête HTTP !
Le dernier PDU est la réponse du serveur. Il n'y a que l'onglet "Inbound" car c'est le client qui reçoit, sans couche sortante. Il n'y a plus de couche "Out" côté client.
Le switch travaille en couche 2 (Liaison). Il lit les adresses MAC (In), décide du port de sortie selon sa table MAC, et réémet (Out). Les couches 3+ ne sont pas traitées — il ne lit pas les IPs.

1. PC-A envoie un ARP Request en broadcast : "Qui est 192.168.1.20 ?"
2. PC-B répond avec un ARP Reply unicast : "C'est moi, ma MAC est XX:XX:XX..."
3. PC-A met à jour sa table ARP (cache)

Ce que tu dois savoir sur les trames beacon et l'association

Les trames beacon sont envoyées périodiquement (~100ms) par le point d'accès pour annoncer sa présence (SSID, BSSID, canaux, débits supportés). Elles permettent aux clients de détecter les réseaux disponibles.
FF:FF:FF:FF:FF:FF — c'est un broadcast. Tout le monde doit pouvoir détecter le réseau WiFi.
Non ! Dans une trame 802.11 en mode infrastructure, l'émetteur (Addr 2) est le point d'accès, mais l'IP source dans le paquet IP est bien celle du terminal mobile. MAC et IP opèrent à des couches différentes.
Ouverte : aucune clé, n'importe qui peut s'associer. Pas de chiffrement.
WPA2 : chiffrement AES, handshake 4 étapes, clé partagée (PSK) ou serveur RADIUS (entreprise). Les données sont illisibles sans la clé.

Comprends ce qui se passe quand Laptop2 se connecte

Source IP : 0.0.0.0 (pas encore d'IP)
Destination IP : 255.255.255.255 (broadcast)
Le message est un DHCP Discover — le client crie "y a-t-il un serveur DHCP ?"
Le PC Admin avait l'IP 192.168.0.x (ancien réseau). Après le changement, le routeur est sur 192.168.50.0/24 — le PC ne peut plus l'atteindre car ils sont sur des réseaux différents.

Routeur WiFi : fait du routage (connecte LAN à Internet), intègre un DHCP, un switch et un AP.
AP seul : étend juste la couverture sans fil — relié par câble au routeur. Pas de routage, pas de DHCP propre. Même SSID/MDP que le routeur principal pour une couverture transparente.

Équipements & Couches

  • Répéteur (L1) : amplifie signal
  • Hub (L1) : broadcast, collisions
  • Switch (L2) : table MAC, full duplex
  • Routeur (L3) : table routage, inter-réseaux

Adressage

  • MAC : 48b · XX:XX:XX:XX:XX:XX
  • Broadcast MAC : FF-FF-FF-FF-FF-FF
  • IPv4 : 32b · Classes A/B/C · CIDR /n
  • IPv6 : 128b · /64 recommandé
  • GUA : publique, 2000::/3
  • LLA : fe80::/10, local

Sous-réseaux IPv4

  • Hôtes = 2^(32-masque) − 2
  • /24 = 254 · /25 = 126
  • /26 = 62 · /27 = 30
  • /28 = 14 · /29 = 6
  • /30 = 2 (liaison P2P)

Modèles

  • OSI 7 couches : Ph·Li·Ré·Tr·Se·Pr·Ap
  • TCP/IP 4 couches : Accès·Internet·Transport·App
  • Encapsulation → Segment → Paquet → Trame → Bits

Ethernet & WiFi

  • Ethernet 802.3 : CSMA/CD
  • WiFi 802.11 : CSMA/CA · RTS/CTS/ACK
