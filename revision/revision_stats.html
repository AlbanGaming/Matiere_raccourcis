Types de variables

Règle pratique

Opérations arithmétiques sans sens → qualitative
Ordre naturel entre modalités → ordinale ; sinon → nominale
Opérations avec sens → quantitative
Toutes les valeurs facilement listables → discrète ; sinon → continue

⚠️
Piège classique : un nombre n'est pas forcément quantitatif. Le code postal 51100 est qualitatif nominal — 51100 − 08200 ne signifie rien géographiquement.
Chapitre 2

Trans­formations des données

📦 Discrétisation ▾

Remplace chaque valeur numérique par l'intervalle (classe) auquel elle appartient. Transforme une variable continue en variable ordinale.

⚠️
Perte d'info irréversible. La moyenne calculée sur les centres de classes diffère de la vraie moyenne. Ex : 9,06 estimé vs 10,36 réel.
🔢 Transformation en rangs ▾

Chaque valeur est remplacée par sa position dans la série triée (de la plus petite à la plus grande). En cas d'ex æquo : rang moyen.

✓
Supprime l'influence des valeurs extrêmes. Garde uniquement l'ordre relatif, pas les distances entre valeurs.
📈 Transformation logarithmique ▾

Remplace chaque valeur x par ln(x) ou ln(1+x). Compresse les grandes valeurs, réduit l'asymétrie droite.

On préfère ln(1+x) car ln(0) est indéfini — utile si x peut valoir 0.

⚖️ Centrage-réduction (normalisation) ▾
z = (x − x̄) / σ

Variable résultante : moyenne = 0, écart-type = 1.

✓
Indispensable pour comparer des variables d'unités différentes et pour les algorithmes d'apprentissage automatique.
Chapitre 3

Distribution groupée

Formules fondamentales
f(m) = n(m) / n (fréquence relative)
Σ f(m) = 1 (somme des fréquences)
F(m) = Σ f(m') pour m' ≤ m (fréquence cumulée)

Mode : modalité (ou classe) d'effectif maximal.

La fréquence cumulée F(m) n'est définie que pour les variables ordinales ou quantitatives (ordre naturel).

Pièges classiques

⚠ Piège 1
Chiffre ≠ Quantitatif

Un code postal, un numéro de téléphone ou un identifiant encodé en chiffres reste qualitatif nominal. La règle : les opérations arithmétiques ont-elles du sens ?

51100 − 08200 = ??? → Pas de sens → Nominal
⚠ Piège 2
r = 0 ≠ "pas de lien"

r = 0 signifie seulement pas de lien linéaire. Un lien non-linéaire parfait (parabole symétrique, cercle…) donne r = 0 !

💡
Toujours observer le nuage de points AVANT de calculer un coefficient.
⚠ Piège 3
Corrélation ≠ Causalité

Corrélation fallacieuse : deux variables peuvent avoir r ≈ 0,99 sans aucun lien causal, juste parce qu'elles partagent une tendance commune (temporelle, géographique…).

Exemple classique : consommation de margarine et taux de divorce dans le Maine (2000–2009) → r ≈ 0,99. Coïncidence temporelle.

⚠ Piège 4
Discrétisation = perte d'info

La moyenne calculée avec les centres de classes peut être très différente de la vraie moyenne sur données brutes.

Vraie moyenne : 10,36 → Après discrétisation : 9,06

N'utiliser les données groupées que si les données brutes ne sont pas disponibles.

⚠ Piège 5
Paradoxe de Simpson

Une tendance observée dans chaque sous-groupe peut s'inverser quand on agrège les données. Cause : une variable de confusion crée un biais.

📌
Ex : les garçons réussissent mieux que les filles dans chaque lycée séparément, mais globalement les filles semblent mieux réussir — car elles sont plus nombreuses dans le lycée plus facile.

→ Toujours chercher les variables de confusion avant de conclure !

⚠ Piège 6
Convention du boxplot

Deux conventions existent. Avant d'interpréter un boxplot, identifier laquelle est utilisée :

Tukey (standard)

Moustaches jusqu'à Q1−1,5×EIQ et Q3+1,5×EIQ
Points isolés = outliers

Min-Max (Excel)

Moustaches jusqu'aux vraies min/max
Pas de points isolés
